Simulink interface

两个接口

Simcenter Amesim-Simulink 和 Simulink-Simcenter Amesim 接口可以结合 Simcenter Amesim 和 Simulink 模型执行仿真。 由于涉及两个软件包，接口提供了两个主要选项：将 Simcenter Amesim 模型导入 Simulink，或将 Simulink 模型导入 Simcenter Amesim。

通过 Simcenter Amesim-Simulink 界面，用户可以选择这两种方法。 除了这两种方法之外，还可以将 Simcenter Amesim 求解器（带有 Simcenter Amesim 模型）导出到 Simulink。 这有时称为协同仿真，因为我们使用两个软件包中的求解器，它们一起执行仿真。

如何选择我需要的接口？

这个问题没有单一的答案。答案取决于很多事情。有些 Simcenter Amesim 系统很难或不可能使用 Simulink 求解器求解。如果是这种情况，那么用户需要选择协同仿真。这意味着将同时使用 Simcenter Amesim 和 Simulink 求解器，并且每个人都将负责自己的部分。

有时 Simulink 系统非常复杂，将其导入 Simcenter Amesim 可能不是最佳解决方案，因为它会强制使用 Simulink 固定步长求解器，这有时并不合适。

除了技术原因之外，还必须考虑模型的用途。如果主要目的是测试/开发 Simcenter Amesim 模型，则将 Simulink 模型导入 Simcenter Amesim 可能是最佳策略。如果主要目的是使用在 Simcenter Amesim 中编写的物理模型在 Simulink 中测试/开发控制器，那么在 Simulink 中工作显然是一个更好的主意。

在许多情况下，一个人开发 Simulink 模型，另一个人开发 Simcenter Amesim 模型。负责整合模型的人显然更喜欢在“他的”环境中工作。

有一些明显的“规则”：如果 Simulink 模型很小，而 Simcenter Amesim 模型很大，则可以将 Simulink 模型放入 Simcenter Amesim。如果需要访问 Simcenter Amesim 中的独特功能，您还可以将 Simulink 模型放入 Simcenter Amesim。如果您需要使用 MATLAB® /Simulink 中的特殊功能，您可以将 Simcenter Amesim 模型放入 Simulink。

Supported Simulink Interface modes

Envisaged interface mode Supported
1.1 Co-simulation in Simcenter Amesim: Simcenter Amesim as Master (Simulink as Slave) \(Yes^{1.2.}\) (SL2AMECosim)
2.1 Co-simulation in Simulink: Simcenter Amesim as Slave (Simulink as Master) \(Yes^{3.}\) (AME2SLCosim)
2.2 Model Exchange: Export of Simcenter Amesim model into Simulink \(Yes^{3.4.}\) (AME2SL)
  1. 对于 SL2AMECosim，请注意，一旦将 Simulink 模型导入 Simcenter Amesim，就不再需要 Simulink 来运行仿真（也不需要 Simulink 许可证）。
  2. 在 SL2AMECosim 中，嵌入式 Simulink 求解器始终为固定步长类型。 支持 Simulink 模型引用。
  3. 对于 Simulink（AME2SL 或 AME2SLCosim）中的模型交换或联合仿真，Simcenter Amesim 模型（带或不带求解器）导出为 S-Function
  4. 此模式符合黑盒选项。

multiple Simulink interfaces

  • 如果 Simcenter Amesim 是目标环境，则接口模块是完全独立的（Simcenter Amesim 将其视为标准子模型）。 因此，在同一个 Simcenter Amesim 草图上混合不同块类型或设置的可能性在理论上是不受限制的。
  • 可以在单个 Simcenter Amesim 系统中使用多个 Simulink 模型（反之亦然），但不能在单个 Simcenter Amesim 系统中多次使用同一个 Simulink 模型（反之亦然）。
  • 目标环境中使用的块与独立系统一样多。
