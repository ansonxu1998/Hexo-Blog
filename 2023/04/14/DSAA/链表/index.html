什么是链表

链表是数据元素的线性集合，其顺序不是由它们在内存中的物理位置给出的。每个元素都指向下一个元素。它是一种数据结构，由一组节点组成，这些节点共同表示一个序列。在其最基本的形式中，每个节点都包含：数据和指向序列中下一个节点的指针（链接）。这种结构允许在迭代期间从序列中的任何位置有效地插入或删除元素。链表的一个缺点是访问时间是线性的(difficult to pipeline)。更快的访问（例如随机访问）是不行的。与链表相比，数组具有更好的缓存局部性(cache locality)。

链表是最简单和最常见的数据结构之一。它们可用于实现其它几种常见的抽象数据类型，包括列表、栈(Stack)、队列、关联数组和 S 表达式，尽管直接实现这些数据结构而不使用链表作为基础并不罕见。

链表相对于传统数组的主要优点是，可以轻松插入或删除列表中的元素，而无需重新分配或重组整个结构，因为数据项不需要连续存储在内存或磁盘上，而在运行时重组数组是一项成本更高的操作。链表允许在列表中的任意位置 插入和删除节点，并且允许在列表遍历期间将链接保留在内存中添加或删除的链接之前，以恒定数量的操作来执行次操作。

但是，由于简单链表本身不允许随机访问数据或任何形式的有效索引，因此许多基本操作（例如获取列表的最后一个节点、查找包含给定基准的节点或定位应插入新节点的位置）可能需要遍历大部分或全部列表元素。

链表的每条记录通常称为“元素”(element) 或节点(node)。包含下一个节点地址的字段通常称为“下一个链接”或“下一个指针”。其余称为“数据”。

为什么要使用链表？

数组(Arrays)可以用来存储相似类型的线性数据，但数组有以下的限制：

  • 数组的大小是固定的：因此使用数组时必须先预设元素数量的上限。此外，通常情况下，无论是否使用，都需要分配数组上限大小的内存。
  • 插入一个元素或删除一个数组中的已有元素花费很大：必须为新元素创建空间，为了创建这个空间，现有的数组元素必须移动。

Example：

在一个系统中，如果我们在一个数组 id[] = [1000, 1010, 1050, 2000, 2040] 中维护一个有序的 ID 列表。此时，想要插入一个新的 ID 1005，那么为了维持排序的顺序，我们必须移动 1000 之后的所有元素。

链表相对于数组的优点

  • 动态数组
  • 易于插入和删除
  • 在开头插入一个元素是一个常数时间操作，需要 O(1) 时间，而在数组的开头插入一个元素需要 O(n) 时间，其中 n 是数组中元素的数量。

链表的缺点

  • 不能随机访问。必须从第一个节点开始按顺序访问节点。因此，不能用它的默认实现对链表进行有效的二分搜索。
  • 对于列表中的每个元素，指针都需要额外的内存空间。
  • 不是缓存友好(cache-friendly)。由于数组元素是连续位置，因此存在引用的局部性，这在链表中是不存在的。
  • 遍历和更改指针需要花费大量的时间。
  • 反向遍历单链表是不可能的。
  • 链表不可能像在数组中那样通过索引直接访问元素。
  • 所搜一个元素的代价很高，需要 O(n) 时间复杂度。
  • 链表的排序是十分复杂的，同时花费巨大。

链表的种类

  • 简单链表 (Simple Linked List) - 只能在一个方向上移动或遍历链表。每个节点的 next 指针指向其它节点，最后一个节点的 next 指针指向 NULL。它也被称为“单链表”。
  • 双链表 (Doubly Linked List) - 在这种类型的链表中看，可以在两个方向（向前和向后）移动或遍历链表。
  • 循环链表 (Circular Linked List) - 链表的最后一个节点的 next 指针指向第一个节点。
  • 双向循环链表 (Doubly Circular Linked List) - 这是一种更复杂的链表类型，它包含指向列表中下一个节点和上一个节点的指针。双链表和循环双向链表的区别，与单链表和循环链表的区别一样的。循环双向链表在第一个节点的前一个字段中不包含 NULL。
  • Header Linked List - 它在列表的开头包含一个标题节点。

链表的应用

  • 链表可以用来实现其它的数据结构，如堆栈和队列
  • 链表可以用来实现哈希表
  • 链表可以用来实现图

链表的操作：Traverse, Insert and Delete

  • Traversal - 访问链表的每一个元素
  • Insertion - 添加一个新的元素到链表中
  • Deletion - 删除一个已有元素
  • Search - 在链表中找到一个节点
  • Sort - 对链表的节点进行排序

遍历一个链表

插入一个元素到链表中

  1. 在开头插入
  • 为新节点分配内存
  • 存入数据
  • 新节点的 next 指向 head
  • 更改 head 指向最新创建的节点
  1. 在末尾插入
  • 为新节点分配内存
  • 存入数据
  • 遍历到最后一个节点
  • 更改最后一个节点的 next 指向最新的节点
  1. 在中间插入
  • 为新节点分配内存
  • 存入数据
  • 遍历到新节点存入位置之前的节点
  • 更改 next 指针

从链表中删除一个节点

  1. 删除头节点
  • 将 head 指向第二个节点
  1. 删除末尾的节点
  • 遍历到倒数第二个元素
  • 将它的 next 指针指向 NULL
  1. 删除中间的节点
  • 遍历到要删除节点之前的节点
  • 更改其 next 指针， 将要删除的节点从链表中排除

在链表中搜索一个元素

  • 将 head 作为 current 节点
  • 运行一个循环，直到 current 节点是 NULL，因为最后一个节点指向 NULL
  • 在每次迭代中，检测节点是否等于 item。如果与 item 匹配，则返回 true，否则返回 false。

链表元素排序

冒泡排序：

  1. 将 head 作为 current 节点，并创建一个 index 节点备用。
  2. 如果 head 是空，则返回。
  3. 否则，运行一个循环直到以后一个节点。
  4. 在每次迭代中依次执行 5-6 步。
  5. 存储 current 的下一个节点在 index 中。
  6. 检查 current 节点的数据是否大于下一个节点的。如果是，则交换 current 和 index。
